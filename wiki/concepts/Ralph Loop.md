# Ralph Loop

## 概述

Ralph Loop 是一个任务监督技能，防止 Claude 在任务未完成时提前收尾，确保工作达到预定标准。

## 核心功能

### 监督机制
- **Hook 拦截**: 捕获退出动作进行检查
- **标准验证**: 检查任务完成标准是否满足
- **继续执行**: 标准未达成时阻止退出并继续工作

### 完成标准要求
- **具体性**: 标准必须明确具体，不能模糊
- **可验证性**: 标准必须能够被客观验证
- **全面性**: 覆盖所有关键完成指标

## 标准示例

### 良好标准
- "接口可用，测试通过，README 补齐 API 文档"
- "所有功能测试通过，代码覆盖率 > 80%，文档完整"
- "部署成功，性能指标达标，监控配置完成"

### 避免的标准  
- "完成开发"（太模糊）
- "差不多了"（不具体）
- "主要功能好了"（不全面）

## 使用价值

### 问题解决
- **提前收尾**: Claude 有时会在任务没完成时提前结束
- **质量不一致**: 完成度判断主观性太强
- **偷停空间**: 模糊标准导致可以提前退出

### 质量提升
1. **标准明确化**: 强制定义具体完成标准
2. **客观验证**: 基于可测量指标判断完成度  
3. **一致性**: 确保所有任务达到相同质量标准

## 实施建议

### 标准制定原则
- **SMART 原则**: 具体、可衡量、可实现、相关、有时限
- **分层标准**: 区分基础要求和进阶要求
- **验收标准**: 明确什么情况下算真正完成

### 集成使用
- 与 [[Planning with Files]] 结合使用
- 作为 [[Superpowers]] TDD 模式的补充
- 用于关键任务的最终验收

## 优势特点

1. **质量保障**: 确保任务真正完成
2. **标准统一**: 避免主观判断差异
3. **防止偷工**: 减少提前结束的可能性
4. **透明性**: 完成标准明确可见
