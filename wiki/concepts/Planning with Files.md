# Planning with Files

## 概述

Planning with Files 是专门为解决长任务中断问题设计的 Claude Code 技能，通过文件持久化来保持任务状态。

## 问题背景

### 长任务挑战
- **上下文压缩**: 复杂任务做久了，对话上下文被压缩
- **记忆丢失**: Claude 容易忘记已经完成的步骤
- **任务中断**: 上下文截断导致工作无法继续

## 解决方案

### 核心方法
- **状态落盘**: 把计划写进文件
- **增量更新**: 每完成一步就更新文件状态
- **恢复机制**: 重新读文件就能接着工作

### 技术实现
- **文件存储**: 中间状态持久化到磁盘
- **状态跟踪**: 记录任务进度和已完成步骤
- **恢复能力**: 从文件重新加载任务上下文

## 工程化价值

### 与传统方法对比
- **会话记忆**: 依赖对话上下文，易丢失
- **文件持久化**: 中间状态落盘，可恢复性强

### 适用场景
- 长时间运行的任务
- 复杂多步骤项目
- 需要中断恢复的工作流

## 使用模式

1. **初始化**: 创建任务计划文件
2. **执行**: 逐步完成任务步骤
3. **更新**: 每完成一步更新文件状态
4. **恢复**: 必要时从文件重新加载

## 优势特点

1. **可靠性**: 不怕上下文丢失
2. **可恢复性**: 任务中断后能继续
3. **透明性**: 进度状态清晰可见
4. **工程化**: 符合软件工程最佳实践
