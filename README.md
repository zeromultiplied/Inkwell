# LLM Knowledge Wiki System

一个由LLM驱动的智能知识库系统，能够持续集成、组织和维护来自各种来源的知识。

## 🌟 项目概述

这是一个基于LLM的知识管理框架，通过三层架构实现知识的持续积累和智能维护：

- **原始层 (/raw)**: 存储原始文档、文章、PDF文件等不可变源材料
- **维基层 (/wiki)**: LLM生成和维护的结构化知识页面
- **输出层 (/outputs)**: 临时性报告、图表和演示文稿

## 🚀 核心特性

- **持续集成**: 自动处理新源文件并集成到现有知识库
- **智能维护**: LLM自动更新引用、标记冲突、维护索引
- **双向链接**: 使用 `[[page-name]]` 语法实现智能导航
- **版本控制**: 完整的Git集成，跟踪知识演变历史
- **Obsidian集成**: 支持在Obsidian中可视化知识图谱

## 📁 项目结构

```
llmwiki/
├── raw/                    # 原始源文件（不可修改）
├── wiki/                   # LLM维护的知识维基
│   ├── concepts/           # 概念和理论页面
│   ├── entities/          # 实体（人物、公司、项目）页面
│   ├── index.md           # 知识索引和导航
│   └── log.md             # 操作日志
├── outputs/               # 临时输出文件
└── AGENTS.md             # 知识代理配置
```

## 🛠️ 使用方法

### 基本命令

```bash
# 处理新的源文件
claude build

# 查看知识库状态
claude status

# 重建索引和导航
claude sync
```

### 添加新内容

1. 将源文件放入 `raw/` 目录
2. 运行 `claude build` 进行处理
3. LLM会自动提取信息、更新相关页面并维护索引

### 查询知识库

- 在对话中直接询问相关知识
- LLM会搜索 `wiki/index.md` 找到相关页面
- 响应中包含引用和来源标记

## 🧩 知识代理

系统支持多个专业代理协同工作：

- **@Librarian**: 负责维护索引和分类系统
- **@Analyst**: 生成深度分析报告
- **@Auditor**: 质量控制，检测逻辑冲突

## 🎯 设计理念

这个系统基于"复合知识"理念：每一次添加新源文件或回答问题都会让整个知识库变得更加丰富。LLM承担了传统知识库维护中的繁琐工作，包括：

- 更新交叉引用
- 保持摘要最新
- 标记数据冲突
- 维护数十个页面的一致性
